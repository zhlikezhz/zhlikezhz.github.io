Unity UGUI之布局系统

UGUI之布局系统

UGUI提供一套组件式的布局系统帮助设计者动态的布局UI元素。这套布局系统有三类关键组件Layout、Fitter和LayoutElement，Layout用来调整挂接在其下的所有的UI元素的大小和位置，Fitter用来调节UI元素的自身大小，而Layout和Fitter都只能作用于拥有LayoutElement组件的UI元素之上。这三类组件结合使用能帮助设计者布局各种各样的UI样式。

Layout

Layout用来布局挂接在其下的所有UI元素，目前UGUI目前提供了三种布局算法：

  • VerticalLayoutGroup：垂直布局，垂直排列UI元素。
  • HorizontalLayoutGroup：水平布局，水平排列UI元素。
  • GridLayoutGroup：网格布局，UI元素网格式排列。

这三个类都继承于LayoutGroup，通过重载SetLayoutHorizontal和SetLayoutVertical来计算和设置其下子UI元素的位置和大小，通过重载CalculateLayoutInputHorizontal和CalculateLayoutInputVertical来计算自身的LayoutElement属性。当每次Layout需要重建的时候，Canvas都会调用这些函数去重建Layout。SetLayoutHorizontal和SetLayoutVertical函数来源于ILayoutController接口，CalculateLayoutInputHorizontal和CalculateLayoutInputVertical函数来源于ILayoutElement接口，用户可以实现它们，以便实现自己的布局算法。

VerticalLayoutGroup、HorizontalLayoutGroup本身都继承于HorizontalOrVeticalLayoutGroup，HorizontalOrVerticalLayoutGroup类CalcAlongAxis函数计算了Layout自身min、preferred、flexible的值以供后续计算子UI的大小位置，而SetChildrenAlongAxis函数计算并设置了子UI的大小和位置。

Layout重建

上面我们分析了Layout提供的一些布局算法，但是我们的布局算法何时会别调用了？调用布局算法的步骤就叫做Layout重建，它由好几个类协同完成。

  • LayoutGroup：提供UI布局算法。
  • LayoutRebuild：调用UI布局算法。
  • CanvasUpdateRegistry：管理整个CanvasUpdate，其中包括Layout重建。

每次LayoutGroup脏的时候，会调用LayoutRebuild.MarkLayoutForRebuild(rectTransform)来标记需要进行Layout重建，LayoutRebuild会将其包装并且加入到CanvasUpdateRegistry的LayoutRebuild队列中，等待下次重建周期，重建周期到来时CanvasUpdateRegistry会逐一调用LayoutRebuild队列中的Rebuild函数来重建Layout，至此Layout重建完成。

Layout脏

一旦LayoutGroup脏了，就会进行重建，那么何时LayoutGroup会脏？

  1. LayoutGroup自身的组件属性被改变。SetProperty()
  2. LayoutGroup的Transform被改变。OnRectTransformDimensionsChange()
  3. LayoutGroup的子元素变更。OnTransformChildrenChanged()
  4. 动画改变了gameObject的属性。OnDidApplyAnimationProperties()
