UGUI 事件系统

UGUI事件系统主要负责将输入事件发送给对象，例如：将用户的键盘、鼠标、触摸等事件分发给对象。

它主要由三部分组成：

  • EventSystem：主要负责处理输入、射线投射以及发送事件。一个场景中只能有一个EventSystem组件，并且需要BaseInputModule组件的协助才能工作。
  • BaseInputModule：是一个基础模块，负责发送输入事件（点击、拖拽、选择等）到具体对象。StandaloneInputModule和TouchInputModule继承自BaseInputModule，分别负责标准输入和触摸输入。我们可以通过继承BaseInputModule来实现自己的输入模块，然后插入到EventSytem中。
  • BaseRaycaster：它通过射线投射来确定输入事件所作用的对象。系统中PhysicsRaycaster、Physics2DRaycaster、GraphicRaycaster都继承自BaseRaycaster来确定对象。 Graphic Raycaster用于UI元素，Physics 2D Raycaster用于2D物理元素，Physics Raycaster用于3D物理元素。

总的来说，EventSystem负责管理，BaseInputModule负责输入，BaseRaycaster负责确定目标对象，目标对象负责处理事件，这样一个完整的事件系统就有了。

EventSystem组件

EventSystem组件继承自UIBehavior，负责管理挂载在其下的BaseInputModule，并保存当前被选中的GameObject。在场景中只需要存在一个这样的管理类，如果场景中出现多个EventSystem组件，Unity会给出警告。

EventSystem每帧激活一个当前平台可用的BaseInputModule类，来处理相应的输入信息，并通知给作用对象。具体的输入信息处理被延迟到BaseInputModule系列类中，具体的作用对象选择被延迟到BaseRaycaster系列类中。而每类事件调用的具体函数在EventExcutes和EventInterfaces中被定义。用户只需要挂接BoxCollider来，并且实现EventInterfaces中定义的相应接口就行了。

BaseInputModule组件

BaseInputModule是用户输入的基类，负责处理用户输入，并将输入事件通知给具体的作用对象。BaseInputModule提供虚函数Process来了处理各种输入信息。

BaseRaycaster组件

射线投射是用于发送确定指示器结束的位置，对于输入模块，常常使用射线投射在场景配置中计算瞄准装置的结束位置。默认设置中提供3种射线投射：

  • Graphic Raycaster 用于UI元素
  • Physics 2D Raycaster 用于2D物理元素
  • Physics Raycaster 用于3D物理元素
    如果你有一个2D/3D射线投射配置在你的场景中，将很容易有非UI元素从输入模块中接受消息。只需要添加实现一个事件接口的脚本。

EventInterfaces.cs文件

此文件中定义了所有可以响应的用户事件接口，继承实现这些接口就可以处理相应的事件。

EventExcutes类

EventExcutes是一个静态的委托类，通过来间接的调用gameObject的响应事件函数。

UntiyEvent类

UnityEvent类是一个委托类，可以通过他来增加委托。它与C#的delegate区别在于，UnityEvent及其子类可以在Unity编辑器中直接编辑关联，非常方便。

UI穿透

处理UI穿透时，可以勾选掉CanvasGroup的Blocks Raycasts参数。这样在属于这个CanvasGroup下的所有事件响应都会被忽略。

响应多个OnDrag

有时候我们在使用ScrollRect时，想要在其上的Item也去想要OnDrag，但是Unity的EventSystem一次只能选择一个作用对象，这时我们可以找Item的OnDrag中去调用其ScrollRect的OnDrag函数来解决此类问题。
